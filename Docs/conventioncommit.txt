# Convention de nommage des commits
```
<type>[optional scope]: <description>

[optional body]
```
___
## Types :
- **feat:** (Une nouvelle feature)
- **fix:** (Bug fix)
- **refactor:** (Un changement de code n'étant pas un fix ni une nouvelle feature)
- **test:** (Ajout ou correction de tests)
- **docs:** (Changements sur la documentation)
- **style:** (Changements esthétiques qui n'affecte pas le code)

## Scope (optionnel) :
- Nom de la feature / fichier affectée.

## Description :
- Titre concis et clair du commit.

## Body (optionnel) : 
- **BREAKING CHANGE:** Si il y a lieu, l'ajout d'une mention BREAKING CHANGE dans le body indique et décrit un changement majeur pouvant entraver le fonctionnement du projet.
- Description plus détaillée du commit.
